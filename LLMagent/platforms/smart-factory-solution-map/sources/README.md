# sources/ 운영 규칙

- 각 리서치 그룹별로 `sources/<그룹>.md`에 누적 기록
- 기록은 `_TEMPLATE.md` 구조를 그대로 사용
- **PPT에는 요약만** 넣고, **근거/세부는 MD에 상세**로 남김
- 추정/루머/2차기사 기반 내용은 `미확인/To-Verify`로 격리하고 PPT에는 미사용

## 파트너십 레벨(4단계, 고정)
- **L1: MOU/발표** — 협력/검토/추진 발표, MOU, 전략적 제휴 선언(= 아직 운영/적용 근거 없음)
- **L2: 계약/파트너십** — 파트너십/계약 체결 + 범위/역할/솔루션 명시(= 적용 여부는 별도)
- **L3: 적용/운영(Go-live)** — 실제 구축/적용/운영/롤아웃(공장/사이트/기간/고객 등 구체)
- **L4: 성과/증빙** — 공시/IR/연차보고서/케이스스터디/레퍼런스 등으로 성과가 확인되는 강한 증거

표기 규칙(권장):
- PPT/MD에서 항목 표기 예: `파트너/솔루션 — 역할 (L3)`
- L1만 있는 항목은 “약함”으로 표시하고, 핵심 맵은 L3~L4 중심으로 채움

## 빠른 시작
1) `_TEMPLATE.md`를 복사해서 `sources/<그룹>.md` 생성
2) 조사할 때마다: MD에 Fact(+Source/Title/Accessed/Quote)부터 쌓기
3) 충분히 쌓이면 PPT에 핵심 bullet만 반영
