# 스마트팩토리 밸류체인 비교(8개 그룹) — 채워진 것 vs 빠진 것 (MD 기반)

> 목적: 각 그룹의 `sources/<그룹>.md`에 **현재까지 출처로 확정된 내용**을 기준으로, 스마트팩토리 밸류체인에서
> - **무엇이 들어가 있는지(근거 있는 항목)**

>
> 주의: 이 문서는 “현재 MD에 적힌 것”의 비교다. 실제로 없는 것이 아니라, **아직 1차 출처로 못 박지 못한 영역**이 ‘비어 있음’으로 표시된다.

## 밸류체인 정의(공통)
- **수집**: 센서/설비/RTLS/IoT/SCADA 등 현장 데이터 취득
- **저장·통합**: 데이터 플랫폼/카탈로그/레이크, MES/ERP/PLM 연동, 표준화
- **분석·AI**: 분석/ML/예지/원인분석/최적화
- **실행·제어**: 공정 제어, 로봇/AGV/자동화, 자동 코딩/엔지니어링
- **운영**: 운영 대시보드/워크플로, 운영지원, 유지보전
- **가상화**: 디지털트윈/시뮬레이션/3D 모델 기반 검증
- **보안**: OT 보안/자산가시성/원격접속/Zone/Conduit 등

## 1) HD현대
**들어간 것(근거 있음)**
- 수집: RTLS 성격의 ‘제품 실시간 위치추적’, 로봇 상태 실시간 감시/예지(HI FACTORY)
- 저장·통합: H!Factory(설비 연결·데이터 취합·분석·관리 + ERP/MES 연동) 서술
- 분석·AI: AI 기반 품질 검사(울산) 언급
- 실행·제어: 용접 로봇/비전 센서/AGV 등 자동화 도입
- 운영: 트윈포스로 공정 상황/정보 확인(업무 효율)
- 가상화: 트윈포스(Twin FOS), 다쏘 버추얼 트윈 협력(MOU)

**빠진 것(추가 출처 필요)**
- 보안: OT 보안(분리/원격접속/자산가시성) 1차 출처 없음
- 저장·통합의 ‘구체 스택’: 어떤 데이터 플랫폼/레이크/표준/마스터데이터 체계인지

## 2) 두산그룹
**들어간 것(근거 있음)**
- 수집: 설비 센서 부착 → 데이터 수집(IoT)
- 분석·AI: 수집 데이터 분석을 통한 공정 최적화(목표/서술)
- 운영: 제조 정보 공유/모니터링(서술)
- 실행(부분): 협력사 지원 프로그램에서 MES/PLM/협동로봇 언급(구체 실행 방식은 아직 약함)

**빠진 것(추가 출처 필요)**
- 저장·통합: MES/PLM/데이터 플랫폼 구조(레퍼런스/구축 사례)
- 가상화: 디지털트윈/시뮬레이션 근거 없음
- 보안: OT 보안 파트너(Claroty/Phosphorus)는 있으나, 제조 밸류체인에 ‘어떻게 적용’됐는지 사례 출처 없음

## 크로스-그룹 관찰(현재 MD 기준)
- **가상화(디지털트윈)**는 HD현대/현대차그룹/LG/포스코에서 강하게 잡히는 반면, 두산/삼성/LS/SK는 아직 약하거나 출처가 부족.
- **보안(OT Security)**는 두산에서 ‘파트너십’은 잡혔지만, 적용 밸류체인/레퍼런스가 비어 있음. 전 그룹 공통으로 ‘보안 레퍼런스’는 부족.
- **SK**는 현재 ‘현장 밸류체인’이 아니라 **AI 인프라/플랫폼(클라우드+GPU+디지털트윈 플랫폼 접근)** 중심으로만 잡힘 → 향후 공장 레퍼런스가 들어오면 급격히 채워질 가능성.
